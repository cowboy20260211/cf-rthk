**文档版本**: 1.0.0  
**编制日期**: 2026-02-22  
**项目名称**: 香港电台 CF 版 (RTHK Radio CF)  
**项目类型**: Web 应用 (在线电台收听平台)

---

## 1. 项目概述

### 1.1 项目背景

香港电台 CF 版是一个基于 Web 的在线收听与节目点播平台，旨在为用户提供便捷的香港电台节目收听体验。该项目解决了用户无法直接在浏览器中访问香港电台网站的问题，通过自建 CORS 代理服务实现数据获取。

### 1.2 项目目标

- 提供香港电台第一台、第二台、第五台的实时直播收听
- 支持节目点播与历史回放功能
- 允许用户收藏喜欢的节目
- 实现跨平台兼容（桌面端、移动端）
- 提供流畅的音频播放体验

### 1.3 目标用户

- 香港地区听众
- 海外华人华侨
- 对香港新闻/文化内容感兴趣的用户

### 2.2 API 接口需求

#### 2.2.1 代理接口

```
GET /api/proxy/{encoded_url}
```

- 功能：通用 CORS 代理
- 用途：获取 RTHK 网站内容
- 缓存：5 分钟
- 超时：15 秒

#### 2.2.2 节目时间表接口

```
GET /api/timetable?channel={channel_id}&date={yyyymmdd}
```

- 功能：获取指定频道指定日期的节目时间表
- 参数：
  - `channel`: 频道 ID (radio1, radio2, radio3, radio4, radio5, pth)
  - `date`: 日期 (YYYYMMDD 格式)

## 3. 非功能需求

### 3.1 性能需求

| 指标             | 目标值  |
| ---------------- | ------- |
| 首屏加载时间     | < 3 秒  |
| 页面切换响应     | < 500ms |
| 音频开始播放时间 | < 2 秒  |
| API 响应时间     | < 2 秒  |

### 3.2 兼容性需求

- Chrome (最新 2 个大版本)
- Firefox (最新 2 个大版本)
- Safari (最新 2 个大版本)
- Edge (最新 2 个大版本)
- iOS Safari (最新 2 个大版本)
- Android Chrome (最新 2 个大版本)

### 3.3 可用性需求

- 移动端触摸友好
- 播放器常驻底部
- 页面刷新后保持播放状态（可选）
- 离线时显示友好提示（可选）

### 3.4 安全需求

- 仅允许 HTTP/HTTPS URL 代理
- 不转发敏感请求头
- 设置合理超时时间
- 错误信息不暴露内部细节

## 4. 技术架构

### 4.1 技术栈

| 层级     | 技术选型                   |
| -------- | -------------------------- |
| 前端框架 | React 18 + TypeScript      |
| 构建工具 | Vite 5                     |
